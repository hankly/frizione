Namespace clutch.xhr

Clutch XML HTTP Request functions. The XML HTTP Request functions are taken from either the standard browser functions or the Gears HttpRequest object, depending on availability. These functions are guaranteed for either the browser or WorkerPool environments.

<static> clutch.xhr.executeRequest(method, url, optionalParams, optionalBody, timeout, handler)
Executes an XHR. The handler function receives three parameters:
  • status - {Number} the status code (can be -1 if the request fails).
  • statusText - {String} the status text.
  • responseText - {String} the response text.
Parameters:
{String} method
can be "GET", possibly "POST".
{String} url
the absolute URL to get or post to.
{Object} optionalParams
optional parameters, do your own value encoding though.
{String} optionalBody
damn useful for posts.
{Number} timeout
the optional maximum amount of time to wait for a reply in milliseconds.
{Function} handler
who to call when things go right, or wrong.
